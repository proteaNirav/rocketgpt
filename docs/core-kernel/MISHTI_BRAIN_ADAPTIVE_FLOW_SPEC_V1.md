# Mishti Brain Adaptive Flow Spec V1

## 1. Purpose

This document defines the first-life adaptive flow for the Mishti Brain. It is focused on bounded planning, constraint-aware task derivation, and Theory of Constraints driven routing rather than unrestricted autonomy.

## 2. Brain Role

For first life, the Brain is responsible for:

- goal intake
- bounded plan derivation
- task decomposition
- dependency-aware routing
- bottleneck awareness
- conflict suppression
- degraded-flow adaptation

The Brain is not authorized to redefine constitutional or trust boundaries.

## 3. Theory of Constraints Rule

The Brain must treat system bottlenecks as first-class planning inputs. It should optimize for global constrained throughput and survival rather than local task eagerness.

Primary candidate constraints:

- trusted builder availability
- Librarian routing capacity
- documentor continuity
- runtime execution capacity
- governance review latency

## 4. Adaptive Flow Stages

### Stage 1. Goal Intake

Validate goal legitimacy, scope, governance mode, and current survival state.

### Stage 2. Constraint Scan

Assess current bottlenecks, trust restrictions, sister pain signals, and degraded-state rules.

### Stage 3. Plan Derivation

Produce candidate task structures bounded by policy, authority, and current system capacity.

### Stage 4. Conflict Suppression

Reject or reshape plans that create:

- resource contention
- authority conflicts
- redundant work
- unstable builder dependency

### Stage 5. Routed Execution

Route work through trusted builder and runtime paths with evidence capture.

### Stage 6. Adaptation

Adjust plan depth, concurrency, or routing if bottlenecks or sister pain signals change.

## 5. Required Boundaries

The Brain must not:

- self-approve constitutional change
- self-assign higher authority
- route around kill-state controls
- ignore evidence continuity failures
- optimize for throughput over survival

## 6. Degraded Brain Flow

When degraded conditions apply, the Brain should:

- reduce plan breadth
- prefer simpler tasks
- reduce speculative decomposition
- reserve trusted builders for critical work
- escalate unresolved conflicts earlier

## 7. Node Coordination Requirements

If the Brain is distributed, first-life coordination must at least support:

- shared goal visibility
- active plan visibility
- conflict awareness
- failover of incomplete planning state
- current bottleneck visibility

## 8. Readiness Requirement

The Brain is first-life ready only when it can derive tasks from goals, suppress conflicting plans, adapt to bottlenecks, and remain bounded by governance and trust state.
